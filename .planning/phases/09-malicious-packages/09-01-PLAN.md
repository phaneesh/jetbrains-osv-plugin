## Implementation

### Detection Layers

The service uses a defense-in-depth approach with four detection layers:

#### Layer 1: Known Malicious Package List
- Hardcoded list of packages known to be involved in malicious incidents
- Fastest check (O(1) hash lookup)
- Examples: `ua-parser-js` (backdoor), `node-ipc` (protestware), `colors` (protestware)

#### Layer 2: OSV API Keyword Analysis
- Queries OSV API for the dependency
- Analyzes vulnerability summaries for malicious keywords
- Keywords: `malicious`, `malware`, `backdoor`, `crypto`, `miner`, `wiper`, `protest`, etc.
- Classifies into [MaliciousType] (MALWARE, BACKDOOR, CRYPTOMINER, PROTESTWARE, etc.)

#### Layer 3: Typosquatting Detection
- Compares package names against a list of popular packages using Levenshtein distance
- Normalized similarity = 1 - distance/maxLength
- Threshold: 0.80 (configurable via [TYPOSQUAT_THRESHOLD])
- Prevents exact matches from being flagged
- Skips if length difference > [MAX_DISTANCE] (performance optimization)

#### Layer 4: Homoglyph / Unicode Attack Detection
- Scans for Cyrillic look-alike characters (`а`, `е`, `о`, `р`, `с`, `х`)
- Detects mixed Latin + Cyrillic scripts
- No legitimate package mixes scripts

### Files Created

1. **`src/main/java/io/dyuti/osvplugin/malicious/MaliciousPackageService.kt`** (~380 lines):
   - `checkPackage(Dependency)`: Single package check with all 4 layers
   - `checkPackages(List<Dependency>)`: Batch check
   - `checkTyposquatting(packageName)`: Returns similarity score + isTyposquat flag
   - `checkHomoglyph(packageName)`: Returns reason string or null
   - Private `levenshteinDistance()`: DP-based edit distance
