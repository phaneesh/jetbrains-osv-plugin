<objective>
Fix broken or non-functional features identified during competitive analysis. These are foundation-level issues that must be resolved before competitive features can be added.

**Features to Fix:**
1. **CVSS Severity Parsing** — OSV API returns severity data but plugin defaults to MEDIUM. Parse CVSSv3/CVSSv2 scores properly.
2. **License Scanning** — UI exists but parser only guesses from package names. Fetch actual license metadata from registries.
3. **Async Inspections** — Current inspections freeze UI on large files. Implement debounced async scanning.
4. **PSI-based Auto-Fix** — Replace string-replacement with PSI-based refactoring for reliable edits.
5. **Batch API Queries** — Fix true batch OSV API queries instead of sequential simulation.

**Why:** Competitive tools (Snyk, Mend, Qodana) all have these basics working correctly. Broken features create a poor impression and prevent adoption.
</objective>

<tasks>
1. <id>06-01-01</id>
   <objective>Fix CVSS severity parsing</objective>
   <action>
   - Update OsVApiService to extract CVSSv3/v2 scores from OSV severity array
   - Add severity enum mapping: 9.0-10.0 → CRITICAL, 7.0-8.9 → HIGH, 4.0-6.9 → MEDIUM, 0.1-3.9 → LOW
   - Update Vulnerability model to store CVSS score
   - Update SeverityUtil to use CVSS when available
   </action>

2. <id>06-01-02</id>
   <objective>Implement real license fetching</objective>
   <action>
   - Create LicenseRegistryService to query:
     - Maven Central REST API for pom.xml license info
     - NPM Registry for package.json license field
     - PyPI JSON API for license classifier
   - Update LicenseParser to use real data instead of string guessing
   - Add SPDX expression parsing (e.g., "MIT OR Apache-2.0")
   - Cache license results in CacheManager
   </action>

3. <id>06-01-03</id>
   <objective>Make inspections async</objective>
   <action>
   - Convert OsVInspection from LocalInspectionTool to use ProgressManager.runInReadActionWithWriteActionPriority
   - Add debouncing (500ms) to prevent scan spam
   - Cache per-file vulnerability results to avoid re-query on every type
   - Show "Scanning..." placeholder while async query runs
   </action>

4. <id>06-01-04</id>
   <objective>Use PSI-based auto-fix</objective>
   <action>
   - Refactor OsVQuickFix to use PsiElement manipulation instead of string replace
   - For Maven: modify XmlTag value via XmlElementFactory
   - For Gradle: modify String literal via KtPsiFactory or GroovyPsiElementFactory
   - For npm: modify JsonProperty value via JsonElementGenerator
   - Wrap changes in WriteCommandAction for undo support
   </action>

5. <id>06-01-05</id>
   <objective>Fix batch API queries</objective>
   <action>
   - Investigate why POST /v1/querybatch returns empty {}
   - Test alternative request formats (purl, different field names)
   - If API issue persists, implement parallel async queries with rate limiting
   - Add fallback: sequential if parallel fails
   </action>
</tasks>
