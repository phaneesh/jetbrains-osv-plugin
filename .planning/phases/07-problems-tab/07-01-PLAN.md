<objective>
Migrate from a custom tool window to the standard JetBrains Problems tool window with a dedicated "OSV Vulnerabilities" tab. This aligns with Mend.io's bundled Package Checker and Qodana patterns, making the plugin feel native to IntelliJ IDEA.

**Technical Approach:**

1. Create a custom ProblemsViewTab using `com.intellij.ui.content.ContentFactory`
2. Implement `com.intellij.codeInsight.daemon.impl.DaemonCodeAnalyzerImpl` integration
3. Replace file-level problem registration with line-level precision
4. Add proper severity icons and navigation
5. Retain the custom tool window for:
   - Project-wide vulnerability dashboard
   - SARIF export
   - Focus mode / branch comparison
   - Advanced filtering

**Why:** Competitive tools (Mend.io, Qodana) integrate with native Problems view. Users expect Alt+6 for security issues. Custom tool windows feel foreign and don't support standard navigation patterns.
</objective>
