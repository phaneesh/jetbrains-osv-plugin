<read_first>

<action>
In OsVConfig.kt, remove the plain-text storage of sensitive tokens and replace with PasswordSafe accessors:

1. Remove from the class body:
   - `var githubToken: String? = null`
   - `var jiraToken: String? = null`
   - `var privacySalt: String? = null`

2. Remove the corresponding assignments from `loadState()`.

3. Add a companion object with PasswordSafe helpers:

```kotlin
companion object {
    private const val GITHUB_TOKEN_KEY = "osv.github.token"
    private const val JIRA_TOKEN_KEY = "osv.jira.token"
    private const val PRIVACY_SALT_KEY = "osv.privacy.salt"

    fun getGithubToken(): String? {
        return try {
            com.intellij.ide.passwordSafe.PasswordSafe.instance
                .getPassword(com.intellij.credentialStore.Credentials(GITHUB_TOKEN_KEY))
        } catch (_: Exception) { null }
    }

    fun setGithubToken(token: String?) {
        try {
            com.intellij.ide.passwordSafe.PasswordSafe.instance
                .setPassword(com.intellij.credentialStore.Credentials(GITHUB_TOKEN_KEY, token))
        } catch (_: Exception) { /* no-op */ }
    }

    fun getJiraToken(): String? {
        return try {
            com.intellij.ide.passwordSafe.PasswordSafe.instance
                .getPassword(com.intellij.credentialStore.Credentials(JIRA_TOKEN_KEY))
        } catch (_: Exception) { null }
    }

    fun setJiraToken(token: String?) {
        try {
            com.intellij.ide.passwordSafe.PasswordSafe.instance
                .setPassword(com.intellij.credentialStore.Credentials(JIRA_TOKEN_KEY, token))
        } catch (_: Exception) { /* no-op */ }
    }

    fun getPrivacySalt(): String? {
        return try {
            com.intellij.ide.passwordSafe.PasswordSafe.instance
                .getPassword(com.intellij.credentialStore.Credentials(PRIVACY_SALT_KEY))
        } catch (_: Exception) { null }
    }

    fun setPrivacySalt(salt: String?) {
        try {
            com.intellij.ide.passwordSafe.PasswordSafe.instance
                .setPassword(com.intellij.credentialStore.Credentials(PRIVACY_SALT_KEY, salt))
        } catch (_: Exception) { /* no-op */ }
    }
}
```

4. Update any call sites that read `config.githubToken`, `config.jiraToken`, or `config.privacySalt` to use `OsVConfig.getGithubToken()`, `OsVConfig.getJiraToken()`, or `OsVConfig.getPrivacySalt()` respectively. Likely call sites: `JiraConnector.kt`, any GitHub advisory integration code, and `PrivacyHasher.kt` or `PrivacyService.kt`.

5. If `OsVConfigurable.kt` has UI fields binding to `config.githubToken`, change them to call `OsVConfig.setGithubToken(...)` on save. Use `ApplicationManager.getApplication()` null-check for safe unit-test execution.
   </action>

- Every call site that previously read `config.githubToken` now calls `OsVConfig.getGithubToken()`

In build.gradle.kts, change the `untilBuild` from `262.*` to `243.*`:
```kotlin
untilBuild.set("243.*")
```

Rationale: The plugin is built against platform 2023.3 (233). Claiming compatibility up to 2026.1 (262._) without testing on 2024.x, 2025.x, or 2026.x is high risk. Narrowing to 2024.3 (243._) provides a safely tested range. This can be expanded in future releases after multi-version testing (Wave 6).
